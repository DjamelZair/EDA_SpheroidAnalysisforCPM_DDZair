CPM Parameter Inference — Output Tables

The following tables document the structured outputs at each step of the six-step RQ3 inference pipeline. These are the exact schemas and example values from the methodology — the discriminability matrix, surrogate validation, nearest-neighbour θ* estimates, and cross-check results. All values shown are illustrative outputs grounded in the methodology doc; final values depend on the simulation database provided by Antonio.

STEP 01 Unit Alignment & Feature Definition
Output table
Feature Conversion Registry
Defines every morphological observable, its formula from the segmentation mask, units after conversion using lattice spacing δ from Antonio, and whether a δ-conversion is required. Dimensionless features bypass all unit error.
Feature Formula from mask Units after conversion δ required? Priority
area Count pixels / sites inside mask μm² (multiply by δ² or px²) Yes Secondary
perimeter Sum contour pixel / site length μm (multiply by δ or px spacing) Yes Secondary
circularity 4πA / P² Dimensionless [0, 1] No — ratio cancels Primary
convexity A / convex hull area Dimensionless [0, 1] No — ratio cancels Primary
eccentricity Fitted ellipse minor / major axes Dimensionless [0, 1] No — ratio cancels Primary
n_fragments Count connected components > min_area Integer count No Primary
frag_area_fraction Fragment area / total area Dimensionless [0, 1] No Primary

Key insight: the four dimensionless features (circularity, convexity, eccentricity, frag_area_fraction) are completely immune to unit errors. A mismatch in those features is guaranteed to reflect a genuine difference in shape, not a scaling error. These are prioritised for calibration robustness until δ is confirmed.

STEP 03 Sensitivity Analysis
Output table — RQ3.2
Discriminability Matrix (W₁)
For each CPM parameter, the simulation database is split into low-value and high-value thirds. The Wasserstein-1 distance between the feature distributions of those two groups is reported. High W₁ = feature is sensitive to that parameter and receives proportionally higher weight in matching. Near-zero W₁ = feature is excluded from the matching criterion for that parameter.
Color: teal = high (>0.25)   amber = medium (0.10–0.25)   muted = low (<0.10)
Parameter circularity n_fragments frag_fraction eccentricity area Verdict
J_cell,cell 0.31 0.28 0.27 0.19 0.04 Informative — all shape features
J_cell,medium 0.38 0.41 0.39 0.22 0.05 Informative — strongest in n_frags
λ_area 0.08 0.06 0.07 0.05 0.44 Area sensitive; shape features not
λ_perimeter 0.12 0.09 0.11 0.08 0.38 Weakly discriminated — limited inferability

Reading this table: J parameters (contact energies) are discriminated by shape features, while λ parameters (constraint strengths) are primarily discriminated by size. The 2D brightfield data is rich in shape information but provides only total projected area for size — meaning λ parameters may be harder to infer than J parameters. A row of near-zero values is a valid scientific result, not a failure.

The dose-response trend across ibrutinib concentrations is the primary biological finding.

Validation output
Internal Consistency Cross-Check
Three checks confirm that θ* is reliable. None require new simulations. All three must pass for the inference to be considered consistent. The example below is for BD22 ibrutinib 1 μM.
Check Value Threshold Interpretation
Surrogate holdout RMSE 0.041 < 0.10 ✓ Surrogate accurate
θ* (Step 5) J_cc=19.2, J_cm=24.3 Reference Nearest-neighbour estimate

Reconstruction W₁ (F_obs vs F̂_sim) 0.052 < 0.10 ✓ F_obs well-reproduced by θ*

Scope note: full visual validation — running CC3D with θ* and comparing the resulting simulation visually to the real spheroid images — is outside the scope of this thesis. That is Antonio's side of the collaboration. The three internal checks above are the validation that is feasible within the thesis itself, and are consistent with best practice in likelihood-free inference (Cranmer et al., 2020).

About this analysis

This EDA pipeline systematically characterises a unique biomedical imaging dataset of Chronic Lymphocytic Leukaemia (CLL) spheroids imaged at multiple timepoints under various drug treatment conditions.

The dataset includes 51 manually annotated brightfield image-mask pairs (ground truth), a time-series pool of VID experiment images with instance segmentation masks, and a raw IncuCyte archive of approximately 102,000 files spanning BTK inhibitor, drug screen, and refractory patient experiments.

Theme 09 extends the EDA into the inverse modelling domain: given segmented spheroid morphology, the pipeline infers which CPM energy parameters best explain each drug condition using Wasserstein nearest-neighbour matching and an XGBoost forward surrogate.

📋
Data Provenance
Images exported from Sartorius IncuCyte S3 live-cell imager. Two data transfers covering BTK, drug-screen, NK, T-cell, and refractory experiment classes.
🧬
Patient Cohort
Peripheral blood CLL patient samples from BD22, BD27, BD35 experiment series. Ibrutinib 0.1/1/10 μM, acalabrutinib 0.1/1 μM, unstimulated control conditions.
📈
Evaluation Metrics
Segmentation: Dice, precision, recall. CPM inference: Wasserstein-1 (W₁), surrogate RMSE, consistency Δ(θ* − θ̂), reconstruction W₁. Sensitivity: discriminability score per feature per parameter.
🛠
Pipeline Stack
Python · NumPy · pandas · matplotlib · scikit-image · scipy · XGBoost · SAM2 · CC3D (CompuCell3D). All outputs reproducible from a single run_all.py call.
